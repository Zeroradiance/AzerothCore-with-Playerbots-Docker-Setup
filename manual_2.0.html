üîå Container Detachment and Log Management

When working with your AzerothCore containers, especially the WorldServer console, proper detachment and log monitoring are essential skills.

How to Safely Detach from Containers

CRITICAL: Never use Ctrl+C to exit from attached containers - this will stop the container! Always use the proper detachment sequence.

Proper Detachment Method

# Attach to WorldServer console docker attach ac-worldserver # Create admin account account create admin password123 account set gmlevel admin 3 -1 # SAFE DETACHMENT: Press Ctrl+P, then Ctrl+Q (NOT Ctrl+C!) # This detaches without stopping the container Ctrl+p Ctrl+q

Detaching from Database and AuthServer Containers

While you typically won't need to attach to database or authserver containers directly, here's how to manage them safely:

# Attach to database container (for direct MySQL access) docker attach ac-database # SAFE DETACHMENT: Use Ctrl+P, then Ctrl+Q Ctrl+p Ctrl+q # Attach to authserver container (for debugging) docker attach ac-authserver # SAFE DETACHMENT: Use Ctrl+P, then Ctrl+Q Ctrl+p Ctrl+q
Note: Database and AuthServer containers typically run as background services. You usually interact with them through commands rather than direct attachment:
  • Database: Use docker exec ac-database mysql -u root -ppassword
  • AuthServer: Monitor logs with docker logs -f ac-authserver

Alternative Detachment Methods

# Attach with custom detach keys docker attach --detach-keys="ctrl-x" ac-worldserver # Now you can detach using Ctrl+X instead # Attach with signal proxy disabled (safer for some containers) docker attach --sig-proxy=false ac-worldserver # Now Ctrl+C will detach instead of stopping the container

üóÉÔ∏è Database Management with Adminer

Adminer provides a web-based interface for managing your AzerothCore databases. This section covers how to connect to your AzerothCore database using an existing Adminer container or creating a new one.

Method 1: Using Existing Adminer Container

If you already have Adminer running on your Unraid system:

# Find your AzerothCore network name docker inspect ac-database | grep NetworkMode # Should show: azerothcore-wotlk_ac-network # Connect your existing Adminer to the AzerothCore network docker network connect azerothcore-wotlk_ac-network your-adminer-container-name

Method 2: Create New Adminer Container

# Navigate to your AzerothCore directory cd /mnt/user/appdata/AzerothCore-with-Playerbots-Docker-Setup/azerothcore-wotlk # Start Adminer connected to AzerothCore network docker run -d --rm --name azerothcore-adminer \ --network azerothcore-wotlk_ac-network \ --link ac-database:db \ -p 8088:8080 \ adminer # Access Adminer at: http://YOUR_UNRAID_IP:8088

Adminer Connection Settings

Field Value Description
System MySQL Database type
Server ac-database Container name (or db if using --link)
Username root Database username
Password password Database password
Database (leave empty) Shows all databases
Quick Access URL: You can bookmark this URL for direct access:
http://YOUR_UNRAID_IP:8088/?server=db&username=root&db=

Available Databases

Your AzerothCore installation includes these databases:

  • acore_auth: User accounts, realmlist, banned accounts
  • acore_characters: Character data, guilds, items, mail
  • acore_world: Game world data, NPCs, quests, loot tables
  • acore_playerbots: Playerbots AI data, configurations, statistics

Common Database Operations

Update Realmlist IP

-- Navigate to acore_auth database -- Find realmlist table -- Edit the address field for id = 1 -- Set to your Unraid IP address UPDATE realmlist SET address = 'YOUR_UNRAID_IP' WHERE id = 1;

Create GM Account

-- In acore_auth database, account table -- Find your account by username -- Note the account ID -- Then in account_access table, add GM privileges INSERT INTO account_access (id, gmlevel, RealmID) VALUES (ACCOUNT_ID, 3, -1);

Check Playerbots Statistics

-- In acore_playerbots database -- Check playerbots_random_bots table for bot list SELECT COUNT(*) as total_bots FROM playerbots_random_bots; SELECT race, class, COUNT(*) as count FROM playerbots_random_bots GROUP BY race, class;

Database Backup via Adminer

Important: Always backup your databases before making changes!
  1. Connect to Adminer using the settings above
  2. Select the database you want to backup
  3. Click "Export" in the left menu
  4. Choose "SQL" format
  5. Select "Data" and "Structure"
  6. Click "Export" to download the backup file

Alternative: Command Line Database Access

# Direct MySQL access via Docker docker exec -it ac-database mysql -u root -ppassword # Access specific database docker exec -it ac-database mysql -u root -ppassword acore_auth # Execute SQL file docker exec -i ac-database mysql -u root -ppassword acore_auth < backup.sql # Create database dump docker exec ac-database mysqldump -u root -ppassword acore_auth > auth_backup.sql

Remove Adminer When Done

# Stop and remove the Adminer container docker stop azerothcore-adminer # The container will be automatically removed due to --rm flag # Or manually remove if needed docker rm azerothcore-adminer

# Get the GUIDs for your AH Bot characters docker exec ac-database mysql -u root -ppassword acore_characters -e "SELECT guid, name, account FROM characters WHERE name LIKE 'Auctionbot%';" # Note down the GUID numbers (e.g., 1012, 1013, 1014)

Step 6: Copy and Configure AH Bot Config File

# Copy config file from container to host docker cp ac-worldserver:/azerothcore/modules/mod-ah-bot/conf/mod_ahbot.conf.dist /mnt/user/appdata/AzerothCore-with-Playerbots-Docker-Setup/wotlk/etc/ # Create working config from template cd /mnt/user/appdata/AzerothCore-with-Playerbots-Docker-Setup/wotlk/etc cp mod_ahbot.conf.dist mod_ahbot.conf # Edit the configuration file nano mod_ahbot.conf

Step 7: Configure AH Bot Settings

Edit these key settings in mod_ahbot.conf:

# Enable AH Bot AuctionHouseBot.EnableSeller = 1 AuctionHouseBot.EnableBuyer = 1 # Set your character GUIDs (replace with your actual GUIDs) AuctionHouseBot.GUIDs = 1012,1013,1014 # Set auction creation rate (600 for fast population) AuctionHouseBot.ItemsPerCycle = 600 # Set minimum and maximum auctions AuctionHouseBot.MinItems = 20000 AuctionHouseBot.MaxItems = 30000 # Enable all crafted items (remove restrictions) AuctionHouseBot.DisabledCraftedItemIDs = ""

Step 8: Copy Config to Correct Docker Location

# Copy configured file to where Docker reads it docker cp /mnt/user/appdata/AzerothCore-with-Playerbots-Docker-Setup/wotlk/etc/mod_ahbot.conf ac-worldserver:/azerothcore/env/dist/etc/modules/ # Also replace the .dist file being read docker cp /mnt/user/appdata/AzerothCore-with-Playerbots-Docker-Setup/wotlk/etc/mod_ahbot.conf ac-worldserver:/azerothcore/env/dist/etc/modules/mod_ahbot.conf.dist

Step 9: Set Characters Online

# Set AH Bot characters to online status (replace GUIDs with yours) docker exec ac-database mysql -u root -ppassword acore_characters -e "UPDATE characters SET online = 1 WHERE guid IN (1012,1013,1014);" # Verify characters are online docker exec ac-database mysql -u root -ppassword acore_characters -e "SELECT guid, name, online FROM characters WHERE guid IN (1012,1013,1014);"

Step 10: Restart and Verify

# Restart WorldServer to apply all changes cd /mnt/user/appdata/AzerothCore-with-Playerbots-Docker-Setup/azerothcore-wotlk docker-compose restart ac-worldserver # Monitor for config loading docker logs ac-worldserver | grep -i "mod_ahbot" | tail -5 # Check auction count (should start growing) docker exec ac-database mysql -u root -ppassword acore_characters -e "SELECT COUNT(*) FROM auctionhouse;"

Troubleshooting AH Bot

If no auctions appear:
  1. Check config loading: Look for "> mod_ahbot.conf" in logs (not .dist)
  2. Verify characters online: All three should show online=1
  3. Check GUIDs: Ensure correct GUIDs in config file
  4. Manual activation: Use .ahbot reload in WorldServer console
